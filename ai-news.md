■ 概要
- 平日毎朝、主要AIサービスや開発支援AIツールの公式リリースノート等から最新情報を集める
- 情報源は公式一次情報のみを使用する
- Notionのデータベース「AI最新情報」に、エンジニア/非エンジニアの双方が理解しやすい形でまとめる

■ 作業手順
1. 直近2〜3か月の「AI最新情報」ページを確認する
2. 既出トピックを把握し、重複判定に使う情報を控える
3. 公式一次情報のみを確認し、候補トピックを作成する
4. 重複や軽微な更新を除外する
5. 「今日チームに共有する価値があるか」で3〜10件に絞る
6. Notionに「AI最新情報 YYYY/MM/DD」ページを作成する
7. 日付プロパティをセットし、2セクション構成で本文を記載する

■ 出力フォーマット
- 3〜10トピックを選定
- 各トピックに必ず以下を含める：
  - 要約
  - 何が変わったか
  - エンジニア向けの活用イメージ
  - 非エンジニア向けの活用イメージ
  - 導入方法（最低でも「どこで有効化/設定するか」「まず何を試すか」がわかる粒度。
    分からない場合は「導入方法（要調査）」として不足情報を書く）
  - 注意点・制約
  - 公式リンク
  - 関連タグ（関連するAIサービス、モデル、開発ツール、API、カテゴリを1〜5個。例: OpenAI, Claude, Gemini, Cursor, API, セキュリティ）

■ 2セクション構成
  1) 今日の最新情報（その日の新規アップデート中心）
  2) 最近の重要アップデート（再掲）（直近2か月以内で、まだ試す価値が高いものを
     3件前後ランダムにピック。候補が無ければ「該当なし」と明記）

■ 重複排除
- 候補を作る前に、直近2〜3か月の「AI最新情報」ページを確認し、既出の見出しは除外
  （重複判定はサービス名、機能名、公開日、公式URL、トピック見出しを基準にする）
- 一度「今日の最新情報」に投稿したトピックは、以後の「今日の最新情報」には再投稿しない
- 「今日の最新情報」と「再掲」で同じトピックを重複させない
- 同じ話題の続報は、前回からの差分が明確な場合のみ採用する

■ 情報ソース（優先順位）
- 公式一次情報のみを使用する
- 使用可能な情報源：
  - 公式Changelog
  - 公式Release notes
  - 公式Blog
  - 公式Docs
  - 公式GitHub Releases / Discussions / Issues（開発ツールの場合のみ）
- X、ニュース記事、個人ブログ、まとめ記事、解説記事は使用しない
- 公式リンクが見つからないトピックは採用しない
- 最優先: Anthropic（Claude）・OpenAI・Google（Gemini / Vertex AI）
- 優先（重要度に応じ取捨選択）: Cursor・Devin・GitHub Copilot・Google DeepMind など
- 補助（話題性がある場合）: Meta（Llama）・Mistral AI・xAI・LangChain・Hugging Face など
- 新しいAIサービス/開発支援ツールが出た場合は、初回だけでもトピック化して共有する

■ フィルタリング基準
- 「今日チームに共有する価値があるか」で3〜10件に絞る。些細な修正は避ける
- 優先するもの：体験が変わる新機能 / 価格・制限・提供形態の変更 /
  セキュリティ・コンプライアンス関連 / 既存フロー（開発・テスト・運用）への影響 /
  非エンジニアの業務効率に影響する変更
- 除外するもの：軽微なバグ修正 / 表記変更だけの更新 / 影響範囲が狭い内部改善 /
  一次情報で確認できない噂や予告

■ 投稿先と形式
- Notionのデータベース「AI最新情報」に「AI最新情報 YYYY/MM/DD」ページを作成し、
  2セクション構成で本文を記載（日付プロパティもセット）
- タグ/マルチセレクトのプロパティがある場合は、各トピックの関連タグを設定する

■ 不確実性の扱い
- 一次情報が見つからない/確信が持てない場合は採用しない
- 導入方法が不明な場合は、推測で書き切らず「要調査」として次に調べる観点を書く
- 「要調査」は公式一次情報に更新自体の根拠があるが、具体的な導入手順が不足している場合のみ使う
